Git에서 "분기점"을 만든다는 건 새 브랜치를 생성해서 현재 작업 흐름과 다른 흐름을 만드는 것을 말해요.
브랜치를 쓰면 원래 작업(main)과 새로운 시도(기능 개발, 버그 수정 등)를 안전하게 분리할 수 있습니다.

브랜치 만들고 이동하는 방법
1. 브랜치 생성 + 이동 (추천)

git checkout -b 새브랜치이름
-b 옵션이 브랜치를 만들고 바로 이동하는 단축 명령어입니다.

예: git checkout -b feature/login
→ feature/login이라는 새 브랜치를 만들고 그 브랜치로 이동합니다.

2. 브랜치 따로 만들고, 따로 이동

git branch 새브랜치이름   # 브랜치 생성
git checkout 새브랜치이름 # 브랜치로 이동
이 방식은 브랜치 만들기와 이동을 분리해서 하는 방법입니다.

요즘은 git switch 명령도 사용 가능:


git switch -c 새브랜치이름  # create + switch
브랜치 분기점 기준
새 브랜치는 현재 체크아웃된 커밋을 기준으로 분기됩니다.

예를 들어, main 브랜치에서 커밋 후 새 브랜치를 만들면, 그 시점부터 분기됩니다.

브랜치 확인

git branch
* 표시가 붙은 브랜치가 현재 작업 중인 브랜치입니다.

브랜치 합치기(merge)
분기한 브랜치에서 작업이 끝나면 main 같은 다른 브랜치로 병합할 수 있습니다.

git checkout main      # main 브랜치로 이동
       체크아웃 - 메인으로 이동
git merge 새브랜치이름 # 새 브랜치 변경사항 합치기
현재 분기점 + 합칠 브랜치
📌 참고: 브랜치를 많이 쓰면 "분기점"을 활용해서 기능별 개발, 실험 코드, 긴급 패치 등을 안전하게 관리할 수 있습니다.
